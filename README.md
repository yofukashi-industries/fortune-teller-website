# 晴道望喜 公式プロフィールサイト

占い師「晴道 望喜（はれみち もちはる）」の個人紹介ホームページです。
HTML / CSS / JavaScript のみで作られており、GitHub Pages でそのまま公開できます。

---

## 1. このサイトについて

- 占い師本人の人柄・占術・鑑定スタイルを紹介する**個人ブランディングサイト**です。
- 予約フォームや決済機能はありません。すべての予約ボタンは、所属店舗（占いの館ノア 名古屋栄店）の**公式予約ページ**へリンクしています。
- スマートフォン表示を最優先に作られています。

---

├── privacy.html        … プライバシーポリシー

│   └── style.css       … デザイン（色・レイアウト）

├── images/

画像は最初は入っていませんが、**画像がなくてもレイアウトは崩れません**（夜明けのグラデーションやプレースホルダーが表示されます）。

## 3. ローカルで確認する方法

1. このフォルダをパソコンに保存します。
2. `index.html` をダブルクリックすると、ブラウザで開けます。
3. 文字や色を変更したら、ブラウザの「再読み込み」で反映を確認できます。

※ 特別なソフトのインストールは不要です。編集にはメモ帳でも使えますが、無料の「Visual Studio Code」がおすすめです。

---

## 4. 占い師名を変更する方法

`index.html` をテキストエディタで開き、`ここを変更` というコメントを目印に、以下を書き換えます。

- `<title>` タグの中
- ヘッダーの `site-logo-name`
- ファーストビューの `hero-name-ja` / `hero-name-en`
- プロフィールの表
- フッター

「晴道 望喜」でページ内検索（Ctrl + F）すると、変更箇所をすべて見つけられます。

---

## 5. キャッチコピーを変更する方法

`index.html` 内の以下の部分を書き換えます。

```html
<!-- ここを変更：キャッチコピー -->
<p class="hero-copy">
  <span>神様から届く言葉を、</span>
  <span>今を生きるあなたの道しるべに。</span>
</p>
```

`<span>` の1つが1行に対応します（PCでは縦書き表示になります）。

---

## 6. プロフィール文章を変更する方法

`index.html` 内の2か所にプロフィール文があります。

- 「晴道望喜について」セクション（`<!-- ここを変更：紹介文 -->` の下）
- 「プロフィール」セクション（`<!-- ここを変更：プロフィール文章 -->` の下）

`<p>` タグの中の文章を書き換えてください。段落を増やすときは `<p>文章</p>` を追加します。

---

## 7. 鑑定料金を変更する方法

`index.html` の「鑑定時間と料金」セクションにあります。

```html
<!-- ここを変更：20分の料金 -->
<span class="price-amount">3,800<span class="price-unit">円</span></span>
```

数字の部分（例：`3,800`）だけを書き換えてください。時間の追加・削除は、`<div class="price-item">` ～ `</div>` のかたまりごとコピー・削除します。

---

## 8. 予約ページURLを変更する方法

予約URLが変わった場合は、**2か所**を変更します。

1. `js/main.js` の最上部
   ```js
   const SITE_LINKS = {
     reservation: "https://noa-fortune.stores.jp/reserve/sakae/2063769#pageContent",
   ```
   この URL を新しいものに書き換えます（サイト内の全予約ボタンに反映されます）。

2. `index.html` などのHTML内の `href="https://noa-fortune.stores.jp/..."`
   JavaScriptが無効な環境でもリンクが動くよう、HTML側にも直接URLが書いてあります。エディタの「すべて置換」機能で、古いURLを新しいURLに一括置換してください。

---

## 9. InstagramなどのSNS URLを変更する方法

`js/main.js` の最上部を書き換えます。

```js
const SITE_LINKS = {
  reservation: "...",
  instagram: "https://instagram.com/ここを変更",
  x: "https://x.com/ここを変更",
  tiktok: "https://www.tiktok.com/@ここを変更",
  youtube: "https://www.youtube.com/@ここを変更",
  line: "https://line.me/R/ti/p/ここを変更"
};
```

`ここを変更` の部分を、実際のアカウントURLに書き換えます。
あわせて、HTML内の同じURL（`href="https://instagram.com/ここを変更"` など）も置換しておくと、JavaScriptが無効な環境でも正しく動きます。

---

## 10. 使用していないSNSを非表示にする方法

`js/main.js` の `SITE_LINKS` で、使わないSNSの値を空文字にします。

```js
tiktok: "",   // ← TikTokを使っていない場合
```

これだけで、サイト内のTikTokカード・フッターのリンクが自動的に非表示になります。

## 11. 画像を差し替える方法

`images` フォルダに、同じファイル名で画像を入れるだけです。

| ファイル名 | 用途 | 推奨サイズの目安 |
|---|---|---|
| `hero.jpg` | ファーストビュー背景 | 横1600px以上 |
| `profile.jpg` | プロフィール写真 | 縦長（4:5）600×750px程度 |
| `og-image.jpg` | SNS共有時の画像 | 1200×630px |
| `favicon.png` | ブラウザタブのアイコン | 正方形 512×512px |

**WebP / AVIF に変更したい場合**：
ファイルを `hero.webp` のように保存し、HTML内の `src="images/hero.jpg"` を `src="images/hero.webp"` に書き換えます。WebP/AVIFはファイルサイズが小さく、表示が速くなります。変換は「Squoosh（https://squoosh.app）」などの無料ツールが使えます。

---

## 12. OGP画像（SNS共有画像）を変更する方法

1. 1200×630px の画像を `images/og-image.jpg` として保存します。
2. `index.html` の `<meta property="og:image" ...>` はそのままで反映されます。
3. 公開後、X や LINE で共有して表示を確認してください（反映に時間がかかる場合があります）。

---

## 13. faviconを変更する方法

正方形のPNG画像を `images/favicon.png` として保存するだけです。

## 14. 配色を変更する方法

`css/style.css` の最上部にある色の設定を書き換えます。

```css
:root {
  --color-bg: #faf8f1;        /* 基本の背景 */
  --color-bg-light: #f2eee1;  /* 交互に使う背景 */
  --color-text: #26272b;      /* 本文の文字色 */
  --color-accent: #a2854a;    /* 金色のアクセント */
  ...
}
```

ここを変えると、サイト全体の色がまとめて変わります。

## 15. GitHubリポジトリを作成する方法

1. https://github.com にアクセスし、アカウントを作成（またはログイン）します。
2. 画面右上の「＋」→「**New repository**」を押します。
3. 「Repository name」にリポジトリ名を入力します（例：`fortune-teller-website`）。
4. 「**Public**」を選択します（GitHub Pagesの無料公開に必要です）。
5. 「Create repository」を押します。

---

## 16. ファイルをGitHubへアップロードする方法

1. 作成したリポジトリのページで「**uploading an existing file**」（または「Add file」→「Upload files」）を押します。
2. このフォルダの中身（`index.html`、`css` フォルダなど）を**すべて**ドラッグ＆ドロップします。
   ※ `fortune-teller-website` フォルダごとではなく、**フォルダの中身**をアップロードしてください。
3. 「Commit changes」を押します。

---

## 17. GitHub Pagesで公開する方法

1. リポジトリの「**Settings**」を開きます。
2. 左メニューの「**Pages**」を開きます。
3. 「Build and deployment」の「Source」で「**Deploy from a branch**」を選択します。
4. Branch で「**main**」、フォルダで「**/ (root)**」を選択します。
5. 「**Save**」を押します。
6. 数分待つと、ページ上部に公開URLが表示されます。

---

## 18. 公開後のURLを確認する方法

公開URLは通常、以下の形式になります。

```
https://あなたのユーザー名.github.io/リポジトリ名/
```

Settings → Pages の画面上部に「Your site is live at ...」と表示されます。

---

## 19. canonical URLを変更する方法

公開URLが決まったら、`index.html` の以下の部分を実際のURLに書き換えます。

```html
<link rel="canonical" href="https://example.github.io/fortune-teller-website/">
```

あわせて、`<meta property="og:url" ...>` と、構造化データ（JSON-LD）内の `"url"` も同じURLに変更してください。

---

## 20. 独自ドメインを設定する場合の概要

1. ドメイン販売サービス（お名前.com、ムームードメインなど）でドメインを取得します。
2. GitHubリポジトリの Settings → Pages → 「Custom domain」にドメインを入力します。
3. ドメイン側のDNS設定で、GitHub Pagesの案内に従って CNAME（またはAレコード）を設定します。
4. 反映後、「Enforce HTTPS」にチェックを入れます。

詳細はGitHub公式ドキュメント「GitHub Pages サイトのカスタムドメインを管理する」を参照してください。

---

## 21. 内容を修正して再公開する方法

1. パソコン上でファイルを修正します。
2. GitHubのリポジトリページで「Add file」→「Upload files」から、修正したファイルをアップロードします（同名ファイルは上書きされます）。
3. 「Commit changes」を押すと、数分でサイトに反映されます。
